React Studio

一屏式 AI CAD 工作台的前端实现

前端围绕 StudioPage 展开，采用左侧输入与模型配置、中央 Three.js 预览、右侧参数与编排信息的布局。用户可以自然语言生成、在线配置模型 Key、导入 3D 文件、上传纹理并实时调参。

前端关键文件

StudioPage.tsx PromptInput.tsx PropertyInspectorPanel.tsx ModelViewer.tsx generationService.ts

页面布局结构

顶部导航

ApexForge 标识、AI 学习手册、AI 文档、GitHub、联系作者二维码弹窗。

左侧面板

Prompt、供应商选择、在线 Key、生成按钮、3D 模型导入、历史记录。

中央预览

Three.js 模型视图、网格地面、OrbitControls、导入资产预览。

右侧面板

属性编辑、生成结果、Agent 编排、技能链、质量评分。

组件职责拆解

StudioPage

保存 viewerParams、selected result、isGenerating、llmProvider、llmApiKeys、history、contactOpen 等页面级状态，并协调生成请求与 UI 更新。

PromptInput

负责 Prompt 输入、多模型供应商选择、在线 API Key 表单和 ModelAssetUploader。导入按钮位于生成按钮下方，整行展示。

PropertyInspectorPanel

提供主题色、强调色、细节级别、复杂度、材质、曲线、连接、纹理上传等可视化控制项。

ModelViewer

根据参数渲染 AI 参数化模型，或解析 importedModelDataUrl 加载 GLB、GLTF、OBJ、STL。

AgentOrchestrationPanel

展示 Prompt Architect、Structure Agent、Surface Detail Agent、Material Agent、Quality Agent 的执行结果。

GenerationPanel / HistoryList

展示当前生成结果、评分、技能链和历史记录，帮助用户回看最近生成内容。

状态流

用户输入 Prompt 后，StudioPage 调用 createLocalGeneration。成功后把返回结果设置为 selected，并把 result.params 合并到 viewerParams；右侧属性编辑直接修改 viewerParams，因此中央 ModelViewer 会立即根据最新参数重建或更新模型。

PromptInput

提交 prompt 与 provider

StudioPage

请求生成并保存状态

viewerParams

统一承载渲染参数

ModelViewer

根据参数重建模型

在线模型配置

UI 入口

PromptInput 的“在线模型配置”区提供 DeepSeek、Kimi、千问三个 password 输入框。

本地保存

StudioPage 使用 apexforge.llmApiKeys 作为 localStorage key，保存当前浏览器的配置。

请求透传

生成时把 llmApiKeys 传入 createLocalGeneration，后端按当前 provider 取对应 Key。

体验兜底

未填写在线 Key 时，后端才尝试读取环境变量；两者都缺失时返回清晰错误提示。

模型导入与图片贴图

3D 模型导入

ModelAssetUploader 使用 FileReader 将 GLB、GLTF、OBJ、STL 转为 dataURL，并写入 importedModelDataUrl、importedModelName、importedModelFormat。

导入优先级

ModelViewer 检测到 importedModelDataUrl 时优先加载用户资产；移除导入模型后恢复 AI 参数化预览。

图片贴图

PropertyInspectorPanel 支持上传图片作为 mainTextureDataUrl，并配置重复次数和纹理强度。

可视化反馈

导入成功后左侧按钮展示文件名；底部视图标签会显示当前是“导入模型资产”还是“AI 参数化模型”。

UI 规范：

项目图标统一使用 lucide-react，顶部 CTA 使用更明显的边框、发光和半透明背景，让 AI 学习手册、AI 文档、GitHub 与联系作者入口更有产品化识别度。
