多模型接入支持 DeepSeek、Kimi、千问，在线 API Key 优先

核心定位

ApexForge 当前的核心不是简单的模型展示器，而是一个完整的 AI 建模体验闭环：自然语言输入、模型供应商选择、在线 Key 配置、后端意图解析、参数化生成、前端三维渲染、可视化属性编辑、导入模型覆盖和历史记录。

Three.js 工厂生成可编辑模型

Studio

预览、调参、导入、贴图、历史沉淀

开发者应该先理解的五件事

Studio 是体验入口

所有核心状态集中在 StudioPage：当前 Prompt、生成结果、Viewer 参数、在线 API Key、历史记录、顶部导航和联系作者弹窗。

后端负责把 Prompt 变成参数

GenerationService 调度 LlmService 与 Harness，产出 ModelGenerationParams、Agent 执行信息、技能链和质量评分。

Three.js 负责最终可视化

ModelViewer 初始化场景、光照、控制器，并根据参数调用 modelFactory 或加载导入资产。

在线 API Key 不写入源码

前端使用 localStorage 保存当前浏览器的供应商 Key，生成时传给后端；后端优先使用在线 Key，未填写才回退环境变量。

默认模型是精致收音机

为了提升初始观感，product 默认分支会生成更完整的复古收音机，但开放品类仍通过关键词进入各自专属模型分支。
