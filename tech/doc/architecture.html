Architecture Blueprint

从用户输入到 3D 模型的全栈架构

ApexForge 采用前后端分层设计：前端负责 Studio 交互、参数编辑和 Three.js 可视化，后端负责 AI 意图解析、Harness 编排、数据持久化和质量校验，数据库记录模板、任务、资产、版本与反馈。

前端分层

层级职责
页面层StudioPage 组合左中右三栏、顶部导航、历史、生成状态、在线 Key、联系作者弹窗。
输入层PromptInput 管理自然语言输入、模型供应商、在线 API Key 和本地 3D 文件导入。
编辑层PropertyInspectorPanel 修改颜色、材质、复杂度、曲线强度、贴图等生成参数。
渲染层ModelViewer 构建 Three.js 场景，调用模型工厂或导入模型加载器。

后端分层

层级职责
Controller暴露 /api/v1/generations、/templates、/feedback、/assets 等接口。
Application ServiceGenerationService 编排任务创建、类别推断、LLM 调用和结果组装。
AI AdapterLlmService 统一 DeepSeek、Kimi、Qwen 的 OpenAI-compatible 调用协议。
PersistencePrismaService 操作 SQLite，保存模板、任务、资产、版本、反馈与质量评分。

核心模块关系

StudioPage

页面级状态中枢，负责把 PromptInput、ModelViewer、PropertyInspectorPanel、AgentOrchestrationPanel、GenerationPanel 与 HistoryList 连接起来。

generationService

前端 API 门面，优先请求后端生成；当网络不可达时，可按约束回退到本地生成逻辑。

GenerationService

后端生成用例入口：推断品类、识别直连关键词、调用 LLM、执行 Harness、保存任务并返回结构化结果。

LlmService

封装多供应商 baseURL、模型名、API Key 选择、系统 Prompt、JSON 解析和错误提示。

ModelViewer

浏览器侧 Three.js 渲染器，管理场景生命周期、导入模型加载、相机控制、阴影和视图标签。

modelFactory

参数化模型工厂，根据 category、silhouette、材质、曲线、连接密度创建可视化模型。

数据模型视角

Prisma schema 将平台数据拆成模板、生成任务、验证报告、质量评分、模型资产、模型版本和用户反馈。这样既能支持当前演示，也为后续版本管理、资产库、用户评价和质量追踪留出扩展点。

Template

预设品类与提示词样板

GenerationTask

一次 AI 建模任务

ModelAsset

导入或生成的模型资产

ModelVersion

资产版本与迭代记录

Feedback

用户评价与改进信号

架构原则：

前端保留高频交互和可视化能力，后端集中处理 AI、数据和质量闭环；模型密钥优先使用用户在线配置，避免把真实密钥固化到源码或文档中。
