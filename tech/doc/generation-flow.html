Generation Pipeline

Prompt 如何变成可渲染的 3D 模型

ApexForge 的生成链路采用“用户意图 → 结构化参数 → Harness 多 Agent → Three.js 参数化模型”的方式。LLM 不直接输出 Three.js 代码，而是输出稳定的建模参数，再交给前端模型工厂渲染。

链路关键词

为什么使用参数协议

直接让大模型生成 Three.js 代码不可控、难验证且安全风险更高。ApexForge 将 AI 输出约束为 ModelGenerationParams，前端只消费白名单字段，并由 modelFactory 生成稳定几何体。这种方式更适合 MVP、在线演示和后续工程化扩展。

{
  "category": "product",
  "bodyColor": "#d4a05f",
  "accentColor": "#2f2f36",
  "materialPreset": "brushed-metal",
  "detailLevel": 8,
  "complexity": 7,
  "silhouette": "复古便携式收音机",
  "edgeStyle": "rounded",
  "curveIntensity": 0.72,
  "connectorDensity": 2
}

Harness 多 Agent 协作

Prompt Architect

分析用户描述，补齐商业级 CAD 约束，锁定建模目标，避免 Prompt 过短造成品类偏移。

Structure Agent

关注主体比例、结构分件、连接关系和可识别轮廓，确保模型不是简单盒子或随机组件。

Surface Detail Agent

补充接缝、格栅、面板、旋钮、纹理区域等可视化细节，提高模型可信度。

Material Agent

根据品类选择金属、陶瓷、玻璃、哑光塑料等材质预设，并形成主体与强调色分层。

Quality Agent

给出质量评分与检查结论，辅助前端展示生成可信度和后续优化方向。

Skill Chain

串联轮廓粗模、硬表面分件、倒角可读性、材质分层和渲染质检。

多供应商与在线 API Key

前端选择供应商

用户在 PromptInput 中选择 DeepSeek、Kimi 或千问，并可直接输入对应 API Key。

浏览器本地保存

API Key 保存到当前浏览器 localStorage，避免要求用户修改项目 .env。

生成时随请求传递

createLocalGeneration 将 llmApiKeys 作为请求体字段发给后端。

后端优先使用在线 Key

LlmService 先读取 apiKeyOverride，未填写时才回退服务端环境变量。

缺失时给出明确提示

如果在线 Key 和环境变量都不存在，后端返回“请在左侧在线模型配置中填写，或在 .env 中设置变量名”的错误。

品类识别与直连关键词

开放品类

生成服务通过 prompt 和 category 推断模型类型，覆盖汽车、手表、首饰、建筑、飞机、家具、镜子、耳机、相机、鞋、包、小家电、容器、机器人、面板设备、收音机等。

直连关键词

对明确品类词进行快速识别，例如“收音机、radio、portable radio、镜子、飞机”等，避免所有未知输入都落入单一默认模型。

默认策略

如果用户没有明确品类，product 默认会生成高质量复古收音机，确保首页初始模型足够美观。

前后端同步

前端本地生成和后端 GenerationService 维护一致的关键词策略，保证断网回退与在线生成体验一致。

关键设计取舍：

LLM 负责理解和规划，Three.js 工厂负责可控渲染。这样既能利用大模型的语义理解，也能避免不可预测代码执行，让模型生成更稳定、更安全、更容易调试。
