## 1. 项目概述

**ApexForge** 是一个基于 Web 的实时 3D 模型生成与展示平台，核心能力是将用户的自然语言描述转化为可交互的 Three.js 模型代码，并在浏览器端高性能渲染。平台聚焦于“程序化生成”路线，采用“固定 HTML 渲染框架 + AI 动态生成 JS 模型代码”的技术范式，完美平衡了生成灵活性、运行性能与开发成本。

**核心差异化：**
- **零后端 3D 计算**：所有渲染在客户端完成，服务端仅处理 AI 推理。
- **代码即模型**：生成结果为标准 Three.js 代码，可编辑、可复用、可审计。
- **企业级安全与稳定**：沙箱执行、代码过滤、自动回退与监控体系。

---

## 2. 业务目标与价值

- **降低 3D 内容创作门槛**：产品经理、设计师可直接描述需求，实时预览原型。
- **加速资产生产流水线**：为游戏、汽车、工业设计提供可批量生成的程序化模型基座。
- **构建可进化的模型知识库**：通过用户交互与参数化模板沉淀，持续优化生成质量。
- **开放 API 生态**：允许第三方通过 REST/WebSocket 集成生成能力。

---

## 3. 系统技术架构概览

**架构说明：**
- **前端**：React SPA + Three.js 渲染引擎，负责交互、渲染、执行生成的 JS 代码。
- **后端**：nestjs + SQLite等。
- **AI 生成服务**：核心微服务，接收描述文本，构建 Prompt，调用大语言模型（如 deepseek），对返回代码进行安全过滤后持久化。
- **模板库**：存储经过人工验证的高质量程序化模板（跑车、飞行器、建筑等），AI 可基于模板进行参数化变体或完全自由生成。

## 4. 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 前端框架 | React 18 + TypeScript | 组件化、类型安全，适合复杂交互 |
| 3D 渲染 | Three.js r160+ | 业界最成熟的 WebGL 库 |

| 后端运行时 | Nestjs | 高并发异步 I/O，与前端技术栈统一 |
| 数据库 | SQLite + JSON/本地文件 |

## 5. 核心模块详细设计

### 5.1 前端框架（ApexForge Studio）

**模块职责：**
- 提供自然语言输入界面，展示生成历史。
- 加载并维护固定的 3D 场景（光照、背景、地面、控制组件）。
- 接收 AI 返回的 JS 代码字符串，在沙箱中执行，将生成的 `THREE.Group` 挂载到场景。
- 提供参数微调面板（如车身颜色、轮毂样式），支持二次生成。

**关键实现：**
- **场景管理器（SceneManager）**：单例，负责场景初始化、灯光、后期处理、轨道控制。对外暴露 `loadModel(group)` 方法。
- **代码执行器（CodeExecutor）**：基于 Web Worker 实现，Worker 中引入 Three.js 库（通过 importScripts 或 module worker），执行代码并返回序列化的模型数据（或直接传递 group 对象给主线程，注意 Transferable 限制）。为保证兼容性，采用 iframe 沙箱方案：在隐藏 iframe 中加载包含 Three.js 的页面，通过 postMessage 传递代码与渲染指令。
- **UI 组件库**：基于 Ant Design / Tailwind，提供 Prompt 输入、历史列表、颜色选择器等。


**内部流程：**
1. 输入校验与敏感词过滤。
2. 构建 System Prompt，注入预选模板代码片段及编码规范。
3. 调用 Deepseek API（或自部署模型），`temperature=0.4` 保证稳定性。
4. 对返回代码进行静态分析：
   - 检查是否包含黑名单 API（`eval`, `fetch`, `WebSocket`, 动态导入等）。
   - 正则提取函数签名，确保符合 `buildModel(color)` 格式。
   - 使用 AST 解析限制循环嵌套层数、几何体创建次数。
5. 若验证失败，自动重试（最多2次），并记录失败日志。
6. 将最终代码存入 PostgreSQL，标记为待人工审核（首次生成）。
7. 通过 WebSocket 实时推送结果给前端，或前端轮询。

**Prompt 设计原则：**
- **角色设定**：资深 Three.js 开发者，擅长程序化建模。
- **强制性约束**：
  - 只能使用 `THREE.BoxGeometry`, `THREE.CylinderGeometry`, `THREE.SphereGeometry`, `THREE.TorusGeometry` 等基础几何体。
  - 材质限定 `MeshStandardMaterial` 或 `MeshPhysicalMaterial`。
  - 车轮必须竖直放置，所有物体开启 `castShadow` 和 `receiveShadow`。
  - 代码必须包含 `return group;`。
- **Few-shot 示例**：提供 2~3 个经典跑车实现，展示正确结构与参数化技巧。

### 5.3 模板库与参数化系统

为解决纯自由生成的造型不可控问题，平台内置一套**分层模型模板**：

- **骨架模板（Skeleton）**：定义车身轮廓 Shape、车轮位置、车窗区域等关键控制点。
- **风格变体（Style Variant）**：基于同一骨架，通过调整贝塞尔曲线控制点、比例、装饰元素产生不同风格。
- **参数暴露接口**：模板代码中包含可替换的变量（如 `bodyLength`, `roofCurvature`），AI 可输出参数对象而非完整代码，极大提升生成速度与质量。


AI 可通过 Function Calling 选择匹配的模板，然后生成参数对象，前端执行：`render(templateId, params)`，而非每次都生成全新代码。

### 5.4 代码执行沙箱

**方案：iframe 完全隔离**
- 创建隐藏 iframe，其 `src` 指向一个预编译的、仅包含 Three.js 核心的静态页面（无其他库）。
- 主线程通过 `postMessage` 向 iframe 发送 `{ type: 'execute', code: '...', color: '#ff0000' }`。
- iframe 内执行代码，获得 `group`，然后使用 `group.toJSON()` 序列化几何体与材质数据，返回给主线程。
- 主线程使用 `THREE.ObjectLoader` 反序列化并挂载到当前场景。
- 优势：完全隔离内存空间，即使代码死循环，也只会阻塞 iframe，可设置超时自动销毁重建。

**安全增强：**
- Content Security Policy (CSP) 仅允许 iframe 执行内联脚本和加载指定 CDN 的 Three.js。
- iframe 页面设置 `sandbox="allow-scripts"`，禁止网络请求、表单提交等。

### 5.5 监控与质量保证

- **AI 生成质量评分**：自动计算模型的面数、对称性、几何体重叠率，将异常结果标记并通知人工审核。
- **用户反馈闭环**：每个生成结果提供“满意/不满意/违规”按钮，反馈数据用于微调 Prompt 或过滤低质模板。
- **全链路追踪**：每个生成请求带 `trace-id`，日志记录从用户输入到代码执行完成的全部耗时。

## 6. 数据流与交互流程

**一次完整的用户生成请求：**
1. 用户在 Studio 输入描述，点击“生成”。
2. 前端发送 HTTP POST 到 `/api/v1/generate`。
3. API 网关鉴权、限流（令牌桶，每用户 10 req/min）。
4. ModelGen Service 接收请求，构建 Prompt，调用 LLM。
5. LLM 返回代码，服务端进行安全校验与格式化。
6. 生成记录写入 DB，状态设置为 `completed`。
7. 通过 Server-Sent Events (SSE) 或 WebSocket 将代码推送至前端。
8. 前端收到代码，转交给沙箱 iframe。
9. iframe 执行代码，返回模型序列化数据。
10. SceneManager 加载模型，替换原有展示模型。
11. 用户可继续通过对话微调（如“把轮毂改成黑色”），前端将当前 Prompt 历史附加上下文重新请求。

## 7. 安全策略

| 层面 | 措施 |
|------|------|
| **代码注入** | 禁止 `eval`、`Function` 在服务端代码外使用；沙箱内执行 AI 代码，且沙箱无网络权限 |
| **LLM 输出过滤** | 正则匹配黑名单关键词；最大 token 限制；对 JS 代码使用 AST 扫描 |
| **API 安全** | JWT 认证 + Rate Limiting；参数输入长度限制（< 2000字符）；防 SQL 注入（ORM） |
| **内容合规** | 对 Prompt 进行敏感词过滤；生成模型不包含品牌标志或违规符号 |
| **基础设施** | K8s NetworkPolicy 隔离微服务；Secret 管理 LLM API Key（Vault） |

## 8. 性能优化

- **前端**：
  - 几何体实例化与 LOD（细节层级），远距离使用低面数模型。
  - Web Worker 进行模型反序列化，避免主线程阻塞。
  - Three.js 使用 `InstancedMesh` 批量渲染重复元素（如轮毂螺丝）。
- **服务端**：
  - 本地缓存库 缓存相同/相似 Prompt 的生成结果（相似度 > 0.95），直接返回。
  - 模板模式下的参数化生成仅需 10～50ms，避免 LLM 调用。
- **网络**：静态资源 CDN 缓存，Gzip/Brotli 压缩，代码增量更新。
