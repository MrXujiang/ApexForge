遵循 NestJS 标准三层结构：`feedback.module.ts` 作为模块装配入口，仅注册 `FeedbackController` 与 `FeedbackService`；`feedback.controller.ts` 暴露 `POST /feedback` 端点，通过 `@Body()` 绑定 DTO；`feedback.service.ts` 封装业务逻辑，先通过 Prisma 查询 `generationTask` 校验关联任务存在（不存在抛 `NotFoundException`），再写入 `feedback` 表，并使用 `createId('fb')` 与 `createTraceId()` 生成 ID 与追踪 ID，统一返回 `{ traceId, data }` 包装体。DTO 层位于 `dto/create-feedback.dto.ts`，使用 class-validator 约束字段类型、枚举值与长度。依赖方向单向：controller → service → PrismaService + common/json 工具，不直接感知外部框架细节。
