- NestJS 模块采用 controller/service 分离，module 仅做 @Module 装配，不承载业务逻辑。
- 对外响应统一包裹 `{ traceId, data }` 结构，traceId 由 `createTraceId()` 生成，便于链路追踪。
- 实体主键通过 `createId(prefix)` 生成（如 `'fb'`），而非依赖数据库自增 ID。
- DTO 校验使用 class-validator 装饰器集中声明，对枚举类字段使用 `@IsIn` 白名单约束。
- 服务层在操作前显式校验外键关联对象存在，缺失时抛出带 code/message 的 `NotFoundException`。
