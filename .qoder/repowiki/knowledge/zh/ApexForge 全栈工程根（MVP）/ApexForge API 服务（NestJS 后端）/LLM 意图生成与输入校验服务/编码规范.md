- 每个功能域拆为 `<name>.module.ts` + `<name>.service.ts` 两文件，Module 仅 `providers`/`exports` 注入对应 Service，不承载业务逻辑。
- 对外部不可信文本统一采用“白名单 fallback + clamp”策略：定义 `fallbackIntent` 常量，用 `normalizeIntent` 合并用户输入并对数值字段执行 `clampNumber(min,max)` 裁剪。
- 错误处理优先抛出 `BadRequestException`，并在 `catch` 分支区分已包装异常与未知异常，同时用 `Logger.warn` 记录原始错误信息。
- 可插拔的外部服务通过 `resolveProviderConfig` 风格的工厂函数根据字符串 provider 返回配置对象，默认值回落到 `deepseek`。
