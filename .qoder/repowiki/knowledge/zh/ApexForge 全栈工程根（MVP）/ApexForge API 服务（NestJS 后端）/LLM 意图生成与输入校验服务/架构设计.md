模块由两个独立的 NestJS 子模块组成：
- `llm/`：`LlmModule` 仅注册并导出 `LlmService`；`LlmService.generateIntent` 通过 `ConfigService` 读取 `DEEPSEEK_*` / `KIMI_*` / `QWEN_*` 环境变量，按 `normalizeProvider` 选择 DeepSeek、Moonshot(Kimi) 或阿里云 DashScope(千问)，以 OpenAI `/chat/completions` 兼容协议发起请求，并用 `extractJson` 从 Markdown fenced 或首尾 `{}` 中抽取 JSON，再经 `normalizeIntent` + `clampNumber` 合并 `fallbackIntent` 做字段归一化与范围裁剪。
- `validation/`：`ValidationModule` 同样只暴露 `ValidationService`；`validatePrompt` 对空串与长度做拦截，`validateCode` 基于 `blockedPatterns` 关键词扫描（eval、Function、fetch、XMLHttpRequest、WebSocket、document、window.top、localStorage、importScripts、require、__proto__）实现 MVP 阶段的 AST 替代方案。
依赖方向单向：上层 Controller → Service → 外部 HTTP/配置，两个模块之间无相互引用，便于独立装配到应用根模块。
