NestJS 标准模块 `GenerationModule`（`generation.module.ts`）作为对外边界，仅暴露 `GenerationService`；内部通过依赖注入组合三个横向子模块：`LlmModule`（调用 deepseek/kimi/qwen）、`TemplatesModule`（按类别选择最佳模板）、`ValidationModule`（prompt/code 校验）。HTTP 层由 `GenerationController` 暴露 RESTful `/generations` 路由，其中 `@SSE(':id/events')` 为前端沙箱渲染预留事件通道（当前返回固定占位消息）。核心编排位于 `GenerationService.create()`：先经 `ValidationService.validatePrompt` 校验 → `inferCategoryFromPrompt` 推断分类 → `TemplatesService.findBestByCategory` 选模板 → `GenerationHarnessService` 生成 `GenerationHarnessPlan`（包含 optimizedPrompt、agents、skillChain、renderProfile），再调用 `LlmService.generateIntent` 产出结构化意图，最后写入 Prisma `generationTask` 并级联创建 `validationReport` 与 `qualityScore`。`GenerationHarnessService` 将“提示词压缩/目标提取/商业约束注入”、“五 Agent 角色编排”和“硬表面分件→倒角→材质分层→质检”的技能链固化为纯函数式 plan 构建器，并通过 `applySkillChain` 对 LLM 输出做确定性增强（clamp 复杂度/面板密度、追加 multi-agent 风格标记）。DTO 使用 class-validator 白名单限制 provider 与 mode，整体依赖方向单向：controller → service → harness/llm/templates/validation/prisma。
