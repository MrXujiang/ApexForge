以 `src/app.module.ts` 为根模块统一 `@Module()` 装配，`src/main.ts` 通过 `NestFactory.create` 启动 Express 实例并挂载全局前缀与中间件；各业务域按 `modules/<domain>` 自包含 Controller/Service/DTO 分层，通过依赖注入在根模块中聚合：
- 基础设施层 `prisma/` 提供 `PrismaModule` + `PrismaService`，被所有持久化模块共享；
- `llm/` 作为外部 LLM 客户端被 `generation/` 调用，同时被 `validation/` 复用做输入安全校验；
- `templates/` 在模块初始化时执行种子 upsert，为 `generation/` 提供参数化模板源；
- `generation/` 编排 LLM 意图解析 → 模板渲染 → 多 Agent 技能链，产出后由 `assets/` 持久化为 ModelAsset/Version；
- `feedback/` 独立记录用户对 GenerationTask 的评分与评论。
跨模块边界仅通过 Nest 依赖注入暴露 Service 接口，DTO 位于各自 `dto/` 子目录并由对应 Controller 使用，无跨模块 DTO 共享。
