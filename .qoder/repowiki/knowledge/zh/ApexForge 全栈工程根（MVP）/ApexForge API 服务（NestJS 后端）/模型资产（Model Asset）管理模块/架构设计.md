遵循 NestJS 标准三层结构：`assets.module.ts` 仅声明 `AssetsController` 与 `AssetsService`；`assets.controller.ts` 暴露 `/assets` 下的 POST/GET/:id/versions 三个路由，职责仅为参数绑定与转发；`assets.service.ts` 承载业务逻辑，通过注入的 `PrismaService` 操作 `modelAsset` 与 `version` 表，并在 `create` 中校验关联的 `generationTaskId` 存在、使用 `createId('asset'/'ver')` 生成 ID、以 `stringifyJson` 写入 tags/metrics/params 等 JSON 字段，再通过本地 `toAssetResult` 把 Date 序列化为 ISO 字符串并解析 JSON 列返回统一 `{ traceId, data }` 响应体。DTO 层 `dto/create-asset.dto.ts` 基于 class-validator 做入参校验。依赖方向单向：controller → service → prisma/common/json，本模块不直接依赖其他业务模块。
