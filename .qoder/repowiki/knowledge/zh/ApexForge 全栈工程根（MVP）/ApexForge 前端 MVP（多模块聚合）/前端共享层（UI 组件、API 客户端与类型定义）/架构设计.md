本模块作为整个前端应用的共享依赖，按职责划分为五个子目录：
- `api/client.ts`：基于原生 `fetch` 的轻量 HTTP 客户端，通过 `VITE_API_BASE_URL` 环境变量注入后端前缀，统一包装响应为 `ApiResponse<T>` 并在非 2xx 时抛出含错误消息的 Error。
- `components/`：遵循 shadcn/ui 风格的原子化 React 组件（Button、Input、Textarea、Card、Alert、Badge），每个组件使用 `class-variance-authority` 声明变体（variant/size），并通过 `@radix-ui/react-slot` 支持 `asChild` 透传根元素；样式通过 `utils/cn.ts` 合并 Tailwind 类名。
- `styles/theme.ts`：集中维护单色主色调（black/graphite/steel/line）与多套模型渲染配色预设（mono/obsidian/champagne/midnight/ruby），供生成结果预览与模板选择使用。
- `types/`：按业务域拆分 TypeScript 接口——`common.ts` 定义跨层通用结构（`ApiResponse`、`AppError`），`generation.ts` 描述模型生成状态机、参数、进度与结果，`template.ts` 承载模板相关契约。
- `utils/`：无副作用纯函数集合，`cn.ts` 封装 `clsx + tailwind-merge`，`format.ts` 处理数值/时间格式化，`validators.ts` 提供表单校验器（返回 null 表示合法）。

依赖方向单向向下：components/utils/types 不互相引用，仅被上层页面消费；api 仅依赖 types/common。
