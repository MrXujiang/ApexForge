本目录按 `src/modules/<feature>` + `src/shared` 的平面结构组织：
- `shared/` 作为唯一的基础设施层，提供 shadcn 风格 UI 组件、`api/client.ts` 请求封装、黑白主题与领域类型，被所有子模块直接引用。
- `modules/studio/` 是主应用入口，组合 `components/` 面板、`pages/StudioPage.tsx` 路由页与 `services/generationService.ts` 编排调用；它消费 `shared/api/client` 发起生成请求，并把结果交给 viewer 渲染。
- `modules/viewer/` 独立封装 Three.js 场景生命周期，通过 `modelFactory.ts` / `modelNormalizer.ts` 将后端模型归一化后渲染，供 studio 以 React 组件形式嵌入。
- `modules/templates/` 提供模板库数据与展示，为 studio 的参数化建模流程提供预设输入。
- `modules/sandbox/` 暴露 `SandboxClient` 与错误映射，作为 studio 与外部沙箱服务的桥接层。
跨模块通信契约集中在 `shared/types/`（`generation.ts`、`template.ts`、`common.ts`），确保各 feature 对同一份领域模型保持一致；UI 样式统一走 `shared/styles/theme.ts` 的黑白主题变量。
