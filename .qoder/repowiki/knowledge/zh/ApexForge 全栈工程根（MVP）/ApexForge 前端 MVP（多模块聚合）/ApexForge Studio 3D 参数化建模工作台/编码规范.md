- UI 组件统一采用 props 回调模式（如 `onParamChange(key, value)`、`onApiKeyChange(provider, value)`）而非内部 state，由父页面集中维护 `ModelGenerationParams` 派生状态。
- 可枚举配置（LLM 提供商、主题色预设、模型类别）以常量数组形式声明在文件顶部，并通过 `.find()` 匹配当前选中项，避免硬编码分支。
- 异步流程使用显式状态机序列（`queued → generating → validating → renderable/failed`），配合 `sleep(ms)` 模拟进度，便于 UI 反馈与错误恢复。
- 对外部服务调用采用 try/catch 包裹并在 TypeError 时回退到本地 mock 数据，保证在无后端环境下仍可运行完整交互原型。
