模块以 `pages/StudioPage.tsx` 为唯一入口，采用三栏布局（左侧建模控制 + 中间 Three.js 预览 + 右侧参数面板），通过集中式 state 管理：`prompt` / `category` / `llmProvider` / `status` / `history` / `selected` 等状态由页面持有并下传给子组件，形成单向数据流。

- `components/` 按职责拆分为输入 (`PromptInput`)、生成状态 (`GenerationPanel`)、历史列表 (`HistoryList`)、属性检查器 (`PropertyInspectorPanel`)、Agent 编排展示 (`AgentOrchestrationPanel`)，均为受控无副作用 UI 组件。
- `services/generationService.ts` 封装后端调用与本地回退策略：优先通过 `@/shared/api/client` 发起 `/generations` 请求，失败时降级到 `createFallbackGeneration` 使用模板数据构造 `GenerationResult`，同时内置 `inferCategoryFromPrompt` 关键词分类器与 `extractOpenCategory` 正则提取器。
- 依赖方向：`StudioPage → components/* + services/* → shared/components + shared/types + modules/templates + modules/viewer`；本模块不直接操作 DOM，仅消费 `ModelViewer` 提供的渲染能力。
- LLM API Key 通过 `apexforge.llmApiKeys` key 持久化在 `localStorage`，生成时随请求体一并下发给后端。
